## Story

**As a** usuario,
**I want** APIs para gerenciar rotinas, executa-las e ver seu status,
**so that** o frontend possa listar, criar, editar, deletar e executar rotinas.

## Acceptance Criteria

1. GET `/api/routines` retorna rotinas ativas com steps, ultima execucao, dependencias e status computado
2. POST `/api/routines` cria rotina com steps e dependencias em transaction
3. GET `/api/routines/[id]` retorna rotina completa com historico de execucoes e dependencias bidirecionais
4. PUT `/api/routines/[id]` atualiza rotina com replace de steps e dependencias
5. DELETE `/api/routines/[id]` remove rotina com cascade
6. POST `/api/routines/[id]/execute` registra execucao com notas opcionais
7. Status computado corretamente: Pendente (nunca executada), OK (dentro do intervalo), Atrasada (intervalo expirado), Desatualizada (dependencia mais recente)
8. Padrao de erro consistente com APIs existentes (apiError, handlePrismaError)

## Dev Notes

### Logica de Status Computado

```
function computeStatus(frequency, customDays, lastExec, depLastExec, today):
  if !lastExec → "Pendente"
  intervalDays = daily:1, weekly:7, monthly:30, custom:customDays
  nextDue = lastExec + intervalDays
  if depLastExec > lastExec → "Desatualizada"  // dependencia foi atualizada depois
  if today > nextDue → "Atrasada"               // passou do prazo
  return "OK"
```

A ordem das verificacoes importa: "Desatualizada" tem prioridade sobre "Atrasada".
