### Two Modes of Operation

[Source: architecture.md#Section 8.2, 8.3]

O `AnalysisView` funciona em DOIS modos:

1. **Wizard mode (client state):** Usado no RecordingPage (Step 5). Edicoes vao para WizardState via dispatch. Itens usam `tempId` (nao sao IDs reais do banco ainda).

2. **Meeting detail mode (API-connected):** Usado no MeetingDetailPage. Edicoes vao para `POST/PUT/DELETE /api/pains` e `/api/solutions`. Itens usam IDs reais do banco.

```typescript
interface AnalysisViewProps {
  // Modo wizard
  mode: 'wizard' | 'detail';
  context?: string | null;
  pains: EditablePain[];         // wizard mode
  dispatch?: React.Dispatch<WizardAction>;  // wizard mode

  // Modo detail (API-connected)
  meetingId?: string;            // detail mode
  dbPains?: Pain[];              // detail mode (com solutions incluidas)

  // Shared
  isLoading?: boolean;
}
```

### EditablePain and EditableSolution Interfaces

### WizardAction Types for Pains

[Source: architecture.md#Section 8.3]

```typescript
type WizardAction =
  | { type: 'UPDATE_PAIN'; index: number; pain: EditablePain }
  | { type: 'ADD_PAIN'; pain: EditablePain }
  | { type: 'REMOVE_PAIN'; index: number }
  // ... outras actions do wizard
```

### tempId Pattern

No modo wizard, itens novos recebem um `tempId` gerado no client (ex: `crypto.randomUUID()` ou `Date.now().toString()`). Esses IDs temporarios sao usados para identificar itens no state local. Quando o usuario salva a reuniao (Story 4.3), os tempIds sao descartados e o backend gera IDs reais com `cuid()`.

### shadcn/ui Components Usados

- **Card**: container para cada dor (com suas solucoes aninhadas)
- **Input**: campos editaveis de descricao (dor e solucao)
- **Button**: adicionar/remover dores e solucoes

### Pains & Solutions API Endpoints (Modo Detail)

[Source: architecture.md#Section 5.2]

| Method | Path | Descricao | Request Body | Response |
|--------|------|-----------|-------------|----------|
| POST | `/api/pains` | Criar dor (vinculada a meeting) | `{description, meetingId}` | `Pain` |
| PUT | `/api/pains/[id]` | Atualizar descricao | `{description}` | `Pain` |
| DELETE | `/api/pains/[id]` | Deletar (cascade solutions) | — | `{success: true}` |
| POST | `/api/solutions` | Criar solucao (vinculada a pain) | `{description, painId}` | `Solution` |
| PUT | `/api/solutions/[id]` | Atualizar descricao | `{description}` | `Solution` |
| DELETE | `/api/solutions/[id]` | Deletar | — | `{success: true}` |
