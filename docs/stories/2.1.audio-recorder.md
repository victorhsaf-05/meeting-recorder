## Story

**As a** usuario,
**I want** gravar audio da reuniao diretamente pelo browser,
**so that** eu nao precise de ferramentas externas.

## Acceptance Criteria

1. Componente `AudioRecorder` com botoes gravar/pausar/parar
2. Timer visivel durante gravacao (mm:ss)
3. Indicador visual de que o microfone esta ativo (waveform simples ou icone pulsante)
4. Audio salvo como Blob em formato WebM/Opus
5. Apos parar a gravacao, permitir playback do audio antes de prosseguir
6. Funciona em Chrome e Firefox modernos
7. Solicita permissao de microfone ao usuario
8. Tratamento de erro se microfone nao estiver disponivel

## Dev Notes

### Hook `useAudioRecorder.ts`

[Source: architecture.md#Section 8.1, 8.3]

O hook encapsula toda a logica do MediaRecorder API:

```typescript
// src/hooks/useAudioRecorder.ts — estrutura conceitual

type RecorderStatus = 'idle' | 'recording' | 'paused' | 'stopped';

interface UseAudioRecorderReturn {
  status: RecorderStatus;
  audioBlob: Blob | null;
  elapsedSeconds: number;
  error: string | null;
  startRecording: () => Promise<void>;
  pauseRecording: () => void;
  resumeRecording: () => void;
  stopRecording: () => void;
  resetRecording: () => void;
}

export function useAudioRecorder(): UseAudioRecorderReturn {
  // 1. getUserMedia({ audio: true }) para obter stream
  // 2. new MediaRecorder(stream, { mimeType: 'audio/webm;codecs=opus' })
  // 3. ondataavailable → acumula chunks
  // 4. onstop → new Blob(chunks, { type: 'audio/webm' })
  // 5. setInterval para timer (pausa/resume acompanha estado)
  // 6. Cleanup: stream.getTracks().forEach(t => t.stop())
}
```

### Component `AudioRecorder.tsx`

[Source: architecture.md#Section 8.1, 8.2]

- Localizado em `src/components/AudioRecorder.tsx`
- Usa shadcn/ui Button e Card
- Step 2 do Wizard (RecordingPage): `AudioRecorder | AudioUpload` (alternativas)
- Audio salvo como Blob no client state (`WizardState.audioBlob`) — NAO persiste no servidor

### MediaRecorder API

- `audio/webm;codecs=opus` e o formato nativo do MediaRecorder em Chrome/Firefox
- Fallback para `audio/webm` se codecs opus nao disponivel
- Verificar suporte: `MediaRecorder.isTypeSupported('audio/webm;codecs=opus')`
- Chrome e Firefox suportam nativamente; Safari tem suporte limitado (fora do escopo MVP)

### Integração com Wizard

[Source: architecture.md#Section 8.2, 8.3]

O componente integra com o wizard via callback:
```
RecordingPage (Step 2)
├── AudioRecorder → onAudioReady(blob) → dispatch({ type: 'SET_AUDIO', blob })
└── AudioUpload   → onAudioReady(blob) → dispatch({ type: 'SET_AUDIO', blob })
```

### Restricoes Tecnicas

[Source: architecture.md#Section 3.1]

- **shadcn/ui v4**: NAO suporta prop `asChild` no Button
- **Audio NAO e persistido no servidor** — apenas mantido em memoria como Blob no client state
- **Sem audioPath no schema** — campo foi removido da arquitetura
