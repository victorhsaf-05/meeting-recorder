## Story

**As a** usuario,
**I want** acessar os detalhes de uma reuniao passada,
**so that** eu possa revisar transcricao, dores e to-dos.

7. Botao "Exportar Excel" para exportar os to-dos da reuniao (componente ExcelExport — implementado na Story 5.2, placeholder aqui)
8. Botao "Deletar reuniao" com confirmacao

### Fetch de Dados

A pagina carrega dados via `GET /api/meetings/[id]` que retorna o Meeting completo com relacoes:

```typescript
// Dados retornados pelo GET /api/meetings/[id]
const meeting = await prisma.meeting.findUnique({
  where: { id },
  include: {
    pains: { include: { solutions: true } },
    todos: true,
    participants: { include: { participant: true } },
  },
});
```

### Componentes Reutilizados

- **TranscriptionView** (Story 2.3): Modo somente leitura — exibe transcricao completa
- **AnalysisView** (Story 3.4): Modo API-connected — editar/criar/deletar dores e solucoes persistem via API
- **TodoTable** (Story 4.2): Modo API-connected — editar/criar/deletar todos persistem via API

### APIs Utilizadas

| Acao | Method | Endpoint |
|------|--------|----------|
| Carregar dados | GET | `/api/meetings/[id]` |
| Atualizar titulo | PUT | `/api/meetings/[id]` |
| Deletar reuniao | DELETE | `/api/meetings/[id]` |
| Editar dor | PUT | `/api/pains/[id]` |
| Deletar dor | DELETE | `/api/pains/[id]` |
| Criar dor | POST | `/api/pains` |
| Editar solucao | PUT | `/api/solutions/[id]` |
| Deletar solucao | DELETE | `/api/solutions/[id]` |
| Criar solucao | POST | `/api/solutions` |
| Editar todo | PUT | `/api/todos/[id]` |
| Deletar todo | DELETE | `/api/todos/[id]` |
| Criar todo | POST | `/api/todos` |

### Delete com Confirmacao

Usar shadcn/ui Dialog para confirmacao.

**NOTA:** `asChild` em `DialogTrigger` e `DialogClose` e suportado normalmente no shadcn/ui v4. A restricao de `asChild` aplica-se APENAS ao componente **Button** diretamente. Os componentes de Dialog do Radix UI continuam funcionando com `asChild`.

```tsx
<Dialog>
  <DialogTrigger asChild>
    <Button variant="destructive">Deletar Reuniao</Button>
  </DialogTrigger>
  <DialogContent>
    <DialogHeader>
      <DialogTitle>Deletar reuniao?</DialogTitle>
      <DialogDescription>
        Esta acao e irreversivel. Todos os dados desta reuniao
        (transcricao, dores, solucoes e to-dos) serao removidos.
      </DialogDescription>
    </DialogHeader>
    <DialogFooter>
      <DialogClose asChild><Button variant="outline">Cancelar</Button></DialogClose>
      <Button variant="destructive" onClick={handleDelete}>Confirmar</Button>
    </DialogFooter>
  </DialogContent>
</Dialog>
```

### Restricoes

- **`'use client'`** obrigatorio (edicao inline, fetch client-side)
- **onDelete: Cascade** no schema garante que deletar reuniao remove tudo automaticamente
- **Re-fetch apos mutacoes**: apos editar dor/solucao/todo, re-buscar dados para garantir consistencia
