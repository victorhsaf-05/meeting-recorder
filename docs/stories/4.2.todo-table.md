### EditableTodo Interface

[Source: architecture.md#Section 4.2]

```typescript
// lib/types.ts

export interface EditableTodo {
  tempId: string;
  action: string;
  responsible: string;
  actionOwner: string;
  costCenter: string;
  account: string;
  deadline: string;
  status: TodoStatus;
  painTempId: string | null;
}

export type TodoStatus = 'Pendente' | 'Em andamento' | 'Concluido' | 'Cancelado';
```

### Dois Modos de Operacao

O componente `TodoTable` opera em dois modos distintos:

1. **Modo Wizard (client state):** Usado na pagina `/recording` (wizard). Os dados vivem apenas no state local (via `useWizard`). Nenhuma chamada API e feita. O array de `EditableTodo[]` e passado como prop e atualizado via callbacks.

2. **Modo API-connected:** Usado na pagina `/meeting/[id]` (detalhes) e no `/dashboard`. Os dados sao carregados via `GET /api/todos` e cada edicao persiste imediatamente via `PUT /api/todos/[id]`. Adicionar linha faz `POST /api/todos`, remover faz `DELETE /api/todos/[id]`.

### Regra de Negocio: Centro de Custo

**CC = CC do Responsavel (demandante), NAO do executor (Resp. pela acao).**

Quando o usuario seleciona um Responsavel no dropdown, o campo Centro de custo deve ser auto-preenchido com o `costCenter` daquele participante. Porem, o campo permanece editavel caso o usuario queira sobreescrever.

### shadcn/ui Components Utilizados

- `Table`, `TableBody`, `TableCell`, `TableHead`, `TableHeader`, `TableRow` — estrutura da tabela
- `Input` — campos editaveis de texto (acao, conta, CC)
- `Select`, `SelectContent`, `SelectItem`, `SelectTrigger`, `SelectValue` — dropdowns (responsavel, status)
- `Calendar` + `Popover`, `PopoverContent`, `PopoverTrigger` — date picker para prazo
- `Badge` — exibir status com cores
- `Button` — adicionar/remover linhas

### Select Guard

[Source: architecture.md#Section 3.1]

shadcn/ui Select `onValueChange` pode retornar null. Sempre usar guard:

```typescript
<Select onValueChange={(v) => v && handleStatusChange(v)}>
  ...
</Select>
```
