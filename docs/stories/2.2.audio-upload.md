## Story

**As a** usuario,
**I want** fazer upload de um audio gravado no Apple Watch ou celular,
**so that** eu possa usar gravacoes feitas fora do browser.

## Acceptance Criteria

1. Botao de upload na mesma tela da gravacao ao vivo
2. Aceita formatos: m4a, mp3, wav, webm
3. Exibe nome e tamanho do arquivo selecionado
4. Validacao de formato (rejeitar tipos nao suportados com mensagem)
5. Preview/play do audio antes de prosseguir (opcional, desejavel)
6. Opcoes de gravar e upload sao alternativas — selecionar uma desabilita a outra

## Dev Notes

### Component `AudioUpload.tsx`

[Source: architecture.md#Section 8.1, 8.2]

- Localizado em `src/components/AudioUpload.tsx`
- Step 2 do Wizard (RecordingPage): alternativa ao `AudioRecorder`
- File armazenado como Blob no client state (`WizardState.audioBlob`) — mesmo campo que o recorder usa

### Restricao shadcn/ui v4 — File Upload Pattern

[Source: architecture.md#Section 3.1]

**CRITICO:** shadcn/ui v4 NAO suporta prop `asChild` no Button. Para file uploads, usar o pattern:

```typescript
// shadcn/ui v4 — usar onClick + hidden input (asChild NAO suportado no Button)
const fileInputRef = useRef<HTMLInputElement>(null);

<Button onClick={() => fileInputRef.current?.click()}>
  Upload Audio
</Button>
<input
  ref={fileInputRef}
  type="file"
  accept="audio/mp4,audio/x-m4a,audio/mpeg,audio/wav,audio/webm"
  className="hidden"
  onChange={handleFileChange}
/>
```

### MIME Types Aceitos

[Source: architecture.md#Section 8.1]

| Formato | MIME Type | Extensao |
|---------|-----------|----------|
| M4A (Apple) | `audio/mp4`, `audio/x-m4a` | .m4a |
| MP3 | `audio/mpeg` | .mp3 |
| WAV | `audio/wav` | .wav |
| WebM | `audio/webm` | .webm |

### Integracao com Wizard

[Source: architecture.md#Section 8.2, 8.3]

O componente integra com o wizard via callback:
```
RecordingPage (Step 2)
├── AudioRecorder → onAudioReady(blob) → dispatch({ type: 'SET_AUDIO', blob })
└── AudioUpload   → onAudioReady(blob, fileName) → dispatch({ type: 'SET_AUDIO', blob, fileName })
```

Opcoes sao mutuamente exclusivas — selecionar uma desabilita a outra no parent (RecordingPage).

### Validacao de Tamanho

[Source: architecture.md#Section 18]

Max file size validation NAO e estritamente necessaria neste componente. O sistema de chunking (Story 2.3) divide automaticamente audios grandes em partes de ~4MB antes de enviar para a API. Portanto, arquivos de qualquer tamanho sao aceitos no upload.
