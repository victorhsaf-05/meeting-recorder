### Mock Helpers — MediaRecorder

```typescript
// src/test/mocks/media.ts

export function mockGetUserMedia() {
  const mockStream = {
    getTracks: () => [{ stop: vi.fn() }],
  } as unknown as MediaStream;

  Object.defineProperty(navigator, 'mediaDevices', {
    value: {
      getUserMedia: vi.fn().mockResolvedValue(mockStream),
    },
    writable: true,
  });

  return mockStream;
}

export function mockMediaRecorder() {
  const MockMediaRecorder = vi.fn().mockImplementation(() => ({
    start: vi.fn(),
    stop: vi.fn(),
    pause: vi.fn(),
    resume: vi.fn(),
    ondataavailable: null,
    onstop: null,
    state: 'inactive',
  }));

  (MockMediaRecorder as any).isTypeSupported = vi.fn().mockReturnValue(true);

  Object.defineProperty(window, 'MediaRecorder', {
    value: MockMediaRecorder,
    writable: true,
  });

  return MockMediaRecorder;
}
```

### TypeScript Config para Testes

Verificar que `tsconfig.json` inclui os tipos do Vitest. Se `globals: true` estiver ativo no vitest.config, adicionar em `tsconfig.json`:

```json
{
  "compilerOptions": {
    "types": ["vitest/globals"]
  }
}
```

Alternativamente, usar triple-slash reference no setup:
```typescript
/// <reference types="vitest/globals" />
```

### Restricoes Tecnicas

- **Vitest** usa Vite internamente — compativel com Next.js via `@vitejs/plugin-react`
- **jsdom** e o environment padrao para testes de componentes React
- **globals: true** evita imports manuais de `describe`, `it`, `expect` em cada arquivo
- **css: false** desabilita processamento de CSS nos testes (performance)
- Testes devem estar em `src/` para acesso aos path aliases `@/`
